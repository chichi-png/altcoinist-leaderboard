## 🔧 Technical Details

### File Structure
```
tools/affiliate-leaderboard/
├── index.html (redirect)
├── leaderboard.html (main app - 220 lines)
├── leaderboard.js (logic - 450 lines)
├── embedded-data.js (data fallback for file:// testing)
├── leaderboard-data.json (18KB - real data)
├── generate-leaderboard-data.py (data generator)
├── styles/
│   ├── groupbot.css (850 lines - design system)
│   ├── leaderboard.css (old - still present but unused)
│   └── mobile-fixes.css (old - still present but unused)
└── affiliate-bios.json (source data)
```

### Data Flow
1. `leaderboard-data.json` ← Generated by `generate-leaderboard-data.py`
2. `generate-leaderboard-data.py` ← Merges:
   - `affiliate-bios.json` (names, handles, links)
   - `leaderboard-data-output.json` (scores from Notion)
3. `leaderboard.js` ← Loads data via fetch() or embedded fallback
4. `embedded-data.js` ← Fallback for file:// protocol testing

### Key Functions (leaderboard.js)
- `loadData()` - Fetch JSON with error handling
- `switchPage(pageName)` - Navigate between pages
- `renderPodium()` - Display top 3
- `renderAlltimeTable()` - Table with pagination
- `renderWeeklyGrid()` - Card grid with pagination
- `renderMonthlyGrid()` - Card grid with pagination
- `renderDirectoryGrid()` - Profile cards with social links
- `openAffiliateModal(rank, source)` - Show affiliate details
- `filterXAffiliates()` - Search functionality (4 functions)
- `sortXAffiliates()` - Sort functionality (4 functions)
- `renderPagination(type, total, perPage)` - Generate pagination
- `goToPage(type, page)` - Handle pagination clicks

### CSS Architecture (groupbot.css)
- **CSS Variables**: Colors, fonts, spacing, transitions
- **Reset & Base**: Normalize styles
- **Typography**: Heading styles, gradient effects, mono labels
- **Layout**: Container, sections, spacing utilities
- **Navigation**: Fixed navbar with backdrop blur
- **Cards**: Multiple card types (feature, stat, affiliate, directory)
- **Tables**: Clean table design with hover effects
- **Buttons**: Primary/secondary with animations
- **Pagination**: Styled with active states
- **Modal**: Backdrop + content with close button
- **Podium**: Winner display with ordering
- **Directory**: Profile cards with social links
- **Responsive**: 3 breakpoints (480px, 768px, 1024px)
- **Utilities**: Margins, text alignment, visibility

## 🐛 Known Issues

### Issue: File Protocol Fetch Error
**Problem**: Browsers block `fetch()` when opening HTML from disk (file:// protocol)
**Impact**: Shows "Failed to Load Data" error
**Solutions Implemented**:
1. ✅ Fallback to embedded data (`embedded-data.js`)
2. ✅ User-friendly error message with reload button
3. ✅ Console logging for debugging

**For Production**: Will work fine on GitHub Pages (https:// protocol)

### Issue: Rank Gaps in Data
**Status**: Intentional (sparse data from Notion)
**Example**: Ranks 1, 5, 7, 8, 9, 10, 13, 14, 16... (non-sequential)
**Impact**: None - this is expected behavior
